Java提高之封装

发表于 2019-04-09 | 分类于 JAVA |

前言

　接触JAVA一年有余,对于JAVA基础从未认真的进行过总结,导致一些基础问题云里雾里,故开篇对JAVA基础做一次总结,本篇为第一篇,JAVA的三大特性之封装;

正篇

封装是什么?

　封装(英语：Encapsulation),从字面上来理解就是包装的意思,而在JAVA中,封装是将一些数据和对该数据的一些操作进行包装后,在内部提供一些对外的操作接口,外部人员可以通过这些接口来对封装内的数据进行操作;

封装能给我们带来哪些好处?

　了解封装能给我们带来哪些好处,好处如下:
　　1. 良好的封装能够减少代码之间的耦合
　　2. 隐藏方法的内部实现,可以让保留调用方法不变的同时,随意修改类的结构,而不影响其他人的运行结果
　　3. 封装还会分开类的属性，将类的属性分成私有属性和公共属性。私有属性仅供类自身调用，和公共属性也仅提供一个供外部调用的方法
　封装可以隐藏对数据操作的具体细节,对外只提供可提供的操作接口;

封装一般在什么时候使用?

　使用情况如下(并不完全):
　　1. 对一些敏感数据的操作,进行封装,只提供接口操作,而不让直接修改;
　　2. 重复的代码,我们将其封装起来,能够对程序的可读性进行提升,对代码的耦合进行降低;

实例

对数据的封装

　举例一: 例如一个用户的信息,一些不太重要的信息,我们可以设置为public,这样对外公开,并不会造成什么影响,而一些隐私的信息,我们希望对其隐藏

　如上为用户的基本信息,对于姓名和年龄,我们可以不做隐藏,但身份证号码和手机号这种隐私信息,是不适合提供外部操作的,所以我们可以通过实例化User类来获取用户的姓名和年龄,但并不提供用户隐私相关的信息给外部人员,在这封装的好处在于一些隐私数据我们将其隐藏.

总结

　封装就好比一件精致的物品,在工厂里它经历了哪些制作过程,你不知道,而当你不知道怎么使用这个物品时,该工厂会在物品的包装里提供一个说明书,该说明书将会指导你怎么使用它,但它的实现原理,你不需要知道;

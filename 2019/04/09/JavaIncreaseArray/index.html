Java提高之数组

发表于 2019-04-09 | 分类于 JAVA |

前言

　接触JAVA一年有余,对于JAVA基础从未认真的进行过总结,导致一些基础问题云里雾里,故开篇对JAVA基础做一次总结,本篇为第四篇,JAVA中的数组;

正篇

数组的定义

　数组是指一组数据的集合，数组中的每个数据称为元素。在Java中，数组也是Java对象。数组中的元素可以是任意类型(包括基本类型和引用类型),但同一个数组里只能存放类型相同的元素,通过数组名加上一个不会越界下标值来唯一确定数组中的元素.C和C++中的数组都可以分配在栈上面,而JAVA中的数组是只能分配在堆上面的,因为JAVA中的数组是引用类型

数组的使用方法

　声明数组,分配空间,赋值和处理.
　声明数组,就是告诉计算机数组的类型是什么
　分配空间,告诉计算机需要给该数组分配多少连续的空间
　赋值,赋值就是在已经分配的空间里面放入数据
　处理,就是对数组元素进行操作,通过数组名[有效下标]来确认数据

数组的声明

　注意：JAVA语言中声明数组时不能指定其长度(数组中的元素个数)
如：int a[5]; 这样声明一维数组是非法的。

数组的模型

　一维数组:就是一行小格. int[] array=new int[10];
　二维数组就是一行加一列组成的一个平面分成的小格. int[][] array=new int[10][10];

数组的静态初始化和动态初始化

　静态初始化:在定义数组的同时就为数组元素分配空间并赋值

　动态初始化

数组的性能

　数组与其它种类的容器之间的区别有三个方面:效率,类型和保存基本类型的能力,在JAVA中,数组是一种效率最高的存储和随机访问对象引用序列的方式,集合类的底层也都是通过数组来实现的.
　在list和数组中对基本类型做求和操作时,由于list.get(i)这个动作是进行拆箱动作,在这方面会过多消耗性能,所有在性能要求较高的场景中请优先考虑数组

数组转换为list注意的地方

　asList返回的是一个长度不可变的列表,数组是多长,转换成的列表是多长,我们无法通过add,remove来增加或者减少其长度

Lambda概念

　lambda表达式是JAVA8中提供的一种新的特性,它支持JAVA也能进行简单的”函数式编程”,它是一个匿名函数,Lambda表达式基于数学中的λ演算得名
　直接对应于其中的lambda抽象,是一个匿名函数,即没有函数名的函数;

Lambda用处以及限制

Lambda用处

　它的用处浅显来说就是替代”内部匿名类”,可以对集合或者数据进行循环操作,减少不必要的代码,使程序逻辑清晰

Lambda的限制

　在可以使用Lambda表达式的地方,方法声明时必须包含一个函数式的接口,任何函数式接口都可以使用Lambda表达式替换
　例如:ActionListener,Comparator,Runnable
　Lambda表达式只能出现在目标类型为函数式接口的上下文中,意味着如果我们提供的这个接口包含一个以上的Abstract Method,那么使用Lambda表达式则会报错;

函数式接口的概念

　函数式接口具有两个主要特征,是一个接口,这个接口具有唯一的一个抽象方法,我们将满足这两个特性的接口称为函数式接口;
　@Functionllnyerface是JAVA8 新加入的一种注解,用于指明该接口类型声明是根据JAVA语言规范定义的函数式接口,JAVA8还声明了一些Lambda表达式可以使用的函数式接口,当你注释的接口不是有效的函数式接口时,可以使@Funcationllnterface解决编译层面的错误;

　当你在类前使用@FunctionalInterface注解时,就将该类设定为一个函数式接口,当你在该类中写入两个抽象方法时,该注解会检查你的类,发现多于一个抽象类或少于一个抽象类时,编译报错,效果见下图;

函数式接口加入Object的接口会怎样?

　当我们向有@FunctionalInterface的接口中放入多个接口时,会有编译错误,但如果我们尝试往接口中放JAVA中的接口时,也会报错,因为这个方法签名是Object类的public方法,@FunctionalInterface并不将它作为一个自己的抽象方法;

　当我们再放一个自己定义的接口时,编译通过;

　但放入Object的clean时,编译又不通过了,因为clean是protected的,由此可以得知,@FunctionalInterface只能接收Object为public的接口,而非public的,该注解会以为是自己定义的接口了;

JAVA提供给我们的SAM接口

Java SE 8中增加了一个新的包:java.util.function,它里面包含了常用的函数式接口,例如:

　Predicate–接收T对象并返回boolean
　Consumer 　–接收T对象,不返回值
　Funcation<T,R>–接收T对象,返回R对象
　Supplier–提供T对象<例如工厂>,不接收值
　UnaryOperator–接收T对象,返回T对象
　BinaryOperator–接收两个T对象,返回T对象

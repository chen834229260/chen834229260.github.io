JAVA 8的新特性之Stream

发表于 2018-05-25 | 分类于 JAVA 8 |

前言

　　学习和了解一下Java 8的新特性;

理论

Stream是什么?

　　1. 流，是一系列数据项，它不是一种数据结构,流可以进行相关的算法和计算，只是它并没有显式地表现，而是在内部进行隐式地操作.
　　2. Stream不是JAVA语言的一部分,它更像一个精心设计的库,充分的利用了lambda表达式
　　3. 流计算的共同结构,它们具有一个流来源,0或多个中间操作,以及一个终止操作;
　　4. 流(集合中的数据)在管道中传输,并且在管道的节点上进行处理,比如筛选,排序,聚合等

Stream能做什么?

　　流是用函数式编程方式在集合类上进行复杂操作的工具，其集成了Java 8中的众多新特性之一的聚合操作，开发者可以更容易地使用Lambda表达式，并且更方便地实现对集合的查找、遍历、过滤以及常见计算等。

集合和流之间的区别

　　1. 流只是在表面上类似于集合,但并不是集合,集合是一种数据结构,它的主要关注点是在内存中组织数据,流的关注点是计算,而不是数据,数据来自其它任何地方
　　2. 不存储数据,流不是一个存储元素的数据结构,它只是传递源的数据
　　3. 功能性的(Functional in nature),在流上操作只是产生一个结果,不会修改源,例如filter只是生成一个筛选后的Stream,不会删除源里的元素
　　4. 流的声明周期内只能访问一次,再次访问只能再重新从源中生成一个stream

流的构成

使用流,分为三个基本步骤:
　　获取数据源(source) → 数据装换(即中间操作,对数据进行过滤,排序等) → 执行操作返回结果(Stream是惰性求值的,当你不需要这个结果时,流是不会将结果计算出来),在该过程中,进行中间操作时,比如进行过滤之后,返回的结果流会是一个新的流,而不是原来我们提供的流进行了过滤。

流的操作类型

　　Intermediate(中间操作)：一个流可以后面跟随零个或多个 intermediate 操作。其目的主要是打开流，做出某种程度的数据映射/过滤，然后返回一个新的流，交给下一个操作使用。

　　Terminal(终止操作)：一个流只能有一个 terminal 操作，当这个操作执行后，流就被使用“光”了，无法再被操作。所以这必定是流的最后一个操作。Terminal 操作的执行，才会真正开始流的遍历，并且会生成一个结果，或者一个 side effect。

Stream的惰性求值

　　Stream在运行过程中不会将需要的结果计算出来,直到你告诉它我需要你的返回结果时,它才会将你需要的结果计算出来,然后将结果返回。

惰性求值(Lazy Evaluation)的好处

　　惰性求值的主要目的是为了减少需要执行的代码量,来提高执行速度,例如在方法调用的时候,在发生实际调用前所传入的参数都会被求值,即使某些参数在方法中根本就没有被用到,因此这就造成了潜在的性能浪费。

Stream的操作

Intermediate(中间操作)：
　　map (mapToInt, flatMap 等)、 filter、 distinct、 sorted、 peek、 limit、 skip、 parallel、 sequential、 unordered

Terminal(终止操作)：
　　forEach、 forEachOrdered、 toArray、 reduce、 collect、 min、 max、 count、 anyMatch、 allMatch、 noneMatch、 findFirst、 findAny、 iterator

Short-circuiting(短路)：
　　anyMatch、 allMatch、 noneMatch、 findFirst、 findAny、 limit

limit/skip

　　limit返回Stream的前面N个元素,skip则是扔掉前N个元素(它是由一个叫sunStream的方法改名而来)

sorted

　　对Stream的排序通过sorted进行,它比数组的排序更强指出在于你可以首先对Stream进行各类map,filter,limit,skip甚至distinct来减少元素数量后,再排序,这样将有效提高执行效率,但也是在有业务需求的时候

min/max/distinct

distinct将Stream中重复的值去除

Match

　　Stream有三个match方法,从语义上来说:
　　allMatch:Stream中全部元素符合传入的predicate,返回true
　　anyMatch:Stream中只要有一个元素符合传入的predicate,返回true
　　noneMatch:Stream中没有一个元素符合传入的predicate,返回true

总结

　　类似与将SQL中的聚合函数进行封装之后,在Java中做成了一个包,这样我们在代码中可以去对数据进行处理,而且因为lambda表达式的惰性求值,并不会产生数据过早加载,导致提前消耗资源,数据只会在需要的时候才去加载,瞬间感觉惰性求值,真的好用…

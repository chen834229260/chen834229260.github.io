每周一篇之Apache-Commons

发表于 2018-09-14 | 分类于 每周一篇 |

前言

　学习项目过程中遇到的一个Apache开源的框架,该框架提供了大量的基础类的封装,类似于一个基础类的工具类.

简述

　记录一下简单的API使用和各个包主要能做什么,由于导包和API过多,不做案例使用,如使用到时,再针对更新.

BCEL

　BCEL是 Java classworking 最广泛使用的一种框架,它可以让您深入 JVM 汇编语言进行类操作的细节。BCEL在实际的JVM 指令层次上进行操作(BCEL拥有丰富的JVM 指令级支持)而Javassist 所强调的源代码级别的工作

BeanUtils

　提供对Java反射和自省API的包装。
依赖包：Commons Codec、Commons Logging

BSF(Bean Scripting Framework)

　Bean Scripting Framework（BSF）是一个支持在Java应用程序内调用脚本语言 (Script)，并且支持脚本语言直接访问Java对象和方法的一个开源项目。有了它 , 你就能在java application中使用javascript, Python, XSLT, Perl, tcl, ……等一大堆scripting language. 反过来也可以，就是在这些scripting language中调用任何已经注册过了的JavaBean,java object。它提供了完整的API实现通过Java访问脚本语言的引擎。

Chain

　Commons Chain实现了Chain of Responsebility和Command模式，其中的Catalog + 配置文件的方式使得调用方和Command的实现方的耦合度大大的降低，提高了灵活性。

CLI

　用java开发命令行工具

Codec

　是编码和解码组件，提供常用的编码和解码方法，如DES、SHA1、MD5、Base64、URL和Soundx等。

Collections

　是一个集合组件，扩展了Java标准Collections API，对常用的集合操作进行了很好的封装、抽象和补充，在保证性能的同时大大简化代码。

Compress

　是一个压缩、解压缩文件的组件，可以操作ar、cpio、Unix dump、tar、zip、gzip、XZ、Pack200和bzip2格式的压缩文件。

Configuration

　是一个Java应用程序的配置管理工具，可以从properties或者xml文件中加载配置信息。
依赖包：Commons Lang、Commons Log、Commons BeanUtils、Commons Collections、Commons Codec

CSV

　是一个用来读写各种Comma Separated Value(CSV)格式文件的Java类库。

Daemon

　Commons Daemon 可以帮你实现将一个普通的 Java 应用变成系统的一个后台服务。例如 Tomcat 就是利用这个项目来实现作为 Linux 和 Windows 的服务启动和停止的。

DBCP

　数据库连接池。
依赖包：Commons Logging、Commons Pool

DbUtils

　是JDBC工具组件，对传统操作数据库的类进行二次封装，可以把结果集转化成List。

Digester

　是XML到Java对象的映射工具集。

Email

　是邮件操作组件，对Java Mail API进行了封装，提供了常用的邮件发送和接收类，简化邮件操作。该组件依赖Java Mail API。
依赖包：Java Mail https://java.net/projects/javamail/pages/Home

Exec

　提供一些常用的方法用来执行外部进程，如执行exe文件或命令行。

FileUpload

　为Web应用程序或Servlet提供文件上传功能，Struts2和SpringMVC的文件上传组件。
依赖包：Commons IO

Functor

　java语言实现函数式编程。

Imaging (previously called Sanselan)

　这是一个用来读写各种图像格式的 Java 类库，包括快速解析图片的基本信息（大小、色彩空间、ICC配置等）和元数据。尽管慢点但非常好的可移植性。比起 ImageIO/JAI/Toolkit 更容易使用，支持更多的格式，提供简单访问元数据的方法。

IO

　是处理IO的工具类包，对java.io进行扩展，提供了更加方便的IO操作。
　工具类包括FileUtils、IOUtils、FilenameUtils和FileSystemUtils，前三者的方法并没有多大的区别，只是操作的对象不同，故名思议：FileUtils主要操作File类，IOUtils主要操作IO流，FilenameUtils则是操作文件名，FileSystemUtils包含了一些JDK没有提供的用于访问文件系统的实用方法。当前，只有一个用于读取硬盘空余空间的方法可用

JCI

　提供通用的Java编译器接口。

JCS

　它是一个复合式的缓冲工具。可以将对象缓冲到内存、硬盘。具有缓冲对象时间过期设定。还可以通过JCS构建具有缓冲的分布式构架，以实现高性能的应用。对于一些需要频繁访问而每访问一次都非常消耗资源的对象，可以临时存放在缓冲区中，这样可以提高服务的性能。而JCS正是一个很好的缓冲工具。缓冲工具对于读操作远远多于写操作的应用性能提高非常显著。

Jelly

　Jexl是对JSP表达式语言的一种扩充，并且不依赖于Servlet API。这就意味着它可以被集成到任何需要表达式语言的应用程序中

Jexl

　Jexl是对JSP表达式语言的一种扩充，并且不依赖于Servlet API。这就意味着它可以被集成到任何需要表达式语言的应用程序中。

JXPath

　提供了使用Xpath语法操纵符合Java类命名规范的 JavaBeans的工具。也支持 maps, DOM 和其他对象模型.

Lang

　是处理Java基本对象方法的工具类包，该类包提供对字符、数组等基本对象的操作，弥补了java.lang api基本处理方法上的不足。

Logging

　提供统一的日志接口，同时兼顾轻量级和不依赖于具体的实现。类包给中间件/日志工具开发者一个简单的日志操作抽象，允许程序开发人员使用不同的具体日志实现工具。

Math

　轻量级自容器的数学和统计计算方法类包，包含大多数常用的数值算法。

Net

　封装了各种网络协议的客户端，支持FTP、NNTP、SMTP、POP3、Telnet等协议。

OGNL

　OGNL表达式语言。

Pool

　提供了一整套用于实现对象池化的框架，以及若干各具特色的对象池实现，可以有效地减少处理对象池化时的工作量。类包用于提高像文件句柄、数据库连接、socket通信这类大对象的调用效率，简单的说就是一种对象一次创建多次使用的技术。

Proxy

　封装了 Java 对象代理的一些常用方法。又叫做 动态代理。

SCXML

　处理SCXML

Validator

　提供了一个简单的、可扩展的框架来在一个XML文件中定义校验器(校验方法)和校验规则。支持校验规则的和错误消息的国际化。

VFS

　提供对不同的文件系统的文件提供统一的操作. 例如ftp,http,本地文件系统等等.

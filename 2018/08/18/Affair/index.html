每周一篇之事务

发表于 2018-08-18 | 分类于 每周一篇 |

前言

　学习事务,了解它的四大特性,即原子性（atomicity）、一致性（consistency）、隔离性（isolation）和持久性（durability）

理论

什么是事务?

　事务简单理解为是指一组操作,在该操作的过程中,单个操作如果失败,则一组操作全部失败,而一组操作全部成功时,才会操作成功;

事务的四大特性

原子性(Atomicity)

　原子性是指一组SQL操作中,要么全部成功执行,否则全部执行失败,进行回滚;

一致性(Consistenct)

　用户A和用户B的前加起来一共是5000; 无论AB用户之间是如何相互转换的，事务结束后两个用户的钱加起来还是5000,这就是事务的一致性

隔离性(Isolation)

　隔离性是指当多个事务来操作数据库时,应该就事务之间相互隔离,防止事物之间的操作数据互相干扰;

持久性(Durability)

　持久性是指事务一旦提交,数据库中对应的数据将会改变为永久性的;

事务不考虑隔离性,会出现的问题

脏读

　脏读指一个事务读取了另外一个事务未提交的数据;

不可重复读

　不可重复读是指在一个事务内,多次读取同一个数据,在这个事务还没有结束,另一个事务也访问该数据,但是由于第二个事务的修改,第一个事务可能读取的是第二个事务已经修改过的数据,导致第一个事务两次读取的数据可能不一样,因此称为不可重复读;

幻读

　当事务不是独立执行时会发生一种现象,例如第一个事务对一个表中的数据进行修改,这种数据涉及到表中的全部数据行,同时,第二个事务野队这个表数据进行修改,这个修改是对表中新增/删除一条数据,那么操作第一个事务的用户发现表中的数据还没有修改的数据行,就好像发生了幻觉,称之为幻读;

脏读,不可重复读和幻读之间的区别

　不可重复的和脏读的区别,脏读是某一个事务读取另一个事务未提交的脏数据,不可重复读则是读取前一事务;
　幻读和不可重复读都读取另一条已经提交的事务,所不同的是不可重复读查询的都是同一数据项,而幻读针对的是一批数据整体;

数据库提供的四种隔离级别:

Read uncommitted(读未提交)

　最低级别,任何情况都会发生;

Read Committed(读已提交)

　可避免脏读的发生;

Repeatable read(可重复读)

　可避免脏读,不可重复的的发生;

Serilizable:

　避免脏读,不可重复读,幻读的发生;

备注

　1. 四种隔离级别,Seralizable级别最高,最低为Read uncommitted,级别越高,执行效率越低,隔离级别的设置只对当前事务有效
　2. Mysql的默认隔离级别是: 可重复读 Repeatable read;
　3. Oracle数据库中,只支持seralizable(串行化)级别和Read committed();默认的是Read committed级别；
　4. 在 MySQL 命令行的默认设置下，事务都是自动提交的，即执行 SQL 语句后就会马上执行 COMMIT 操作。因此要显式地开启一个事务务须使用命令 BEGIN 或 START TRANSACTION，或者执行命令 SET AUTOCOMMIT=0，用来禁止使用当前会话的自动提交。

当事务级别为read uncommitted时，会引发脏读，不可重复的和虚读

事务隔离级别为read committed时，会引发不可重复读和续读，但避免了脏读

当事务级别为repeatable read时，会引发续读，但避免了脏读和不可重复读

当事务级别为Serializable时，会避免所有问题

Spring Boot中事务的注解方式

@Transactional注解的属性介绍

Value和transactionManager属性

　作用一致,当配置了多个事务管理器时,可以使用该属性指定选择哪个事务管理器

propagation属性

　事务的传播行为,默认值为Propagation.REQUIRED

  1. Propagation.REQUIRED: 如果当前存在事务,则加入该事务,如果当前不存在事务,则创建一个新事务
  2. Propagation.SIPPORTS: 如果当前存在事务,则加入该事务,如果当前不存在事务,则以非事务的方式继续运行
  3. Propagation.MANDATORY: 如果当前存在事务,则加入该事务,如果当前不存在事务,则抛出异常
  4. Propagation.REQUIRES_NEW: 重新创建一个新的事务,如果当前存在事务,暂停当前的事务
  5. Propagation.NOT_SIPPORTED 以非事务的方式运行,如果当前存在事务,暂停当前的事务
  6. Propagation.NEVER 以非事务的方式运行,如果当前存在事务,则抛出异常
  7. Propagation.NESTED 和Propagation.REQUIRED效果一样

timeout属性

　事务的超时实践,默认为-1,如果改时间事务没有完成,则事务自动回滚

readOnly属性

　指定事务是否为只读,默认为false

rollbackFor属性

　用于指定能够触发事务回滚的异常类型,可以指定多个异常类型

注意点

  1. 使用@Transaction的方法,对同一个类里面的方法调用,@Transactionl无效,比如有一个类Test，它的一个方法A，A再调用Test本类的方法B（不管B是否public还是private），但A没有声明注解事务，而B有。则外部调用A之后，B的事务是不会起作用的。
  2. 在action中加上@Transactional，不会回滚。切记不要在action中加上事务
  3. 默认配置下spring指挥回滚运行时,未检查异常或者error
  4. @Transactional注解只能应用到public方法才有效
  5. Spring boot会自动配置一个DataSourceTransactionManager,所以在方法或者类上加@Transactional注解,就会自动纳入spring的事务管理
